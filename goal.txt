## 3. 执行步骤

### Phase 1: 启动本地 Android 模拟器

**方式 A：Android Studio AVD Manager**
1. 打开 Android Studio → More Actions → Virtual Device Manager
2. 创建/启动 `Pixel_8_Pro_API_35`（首次启动需先下载 system image）
3. 等系统完全启动到桌面（看到"Hello"或 Launcher）

**方式 B：命令行（无 Studio）**
```bash
# 列出现有 AVD
$ANDROID_HOME/emulator/emulator.exe -list-avds
# 启动 headless（适合 CI / 沙箱）
$ANDROID_HOME/emulator/emulator.exe -avd Pixel_8_Pro_API_35 -no-window -no-audio &
sleep 60   # 等系统完全启动
adb wait-for-device   # 阻塞直到设备就绪
adb shell getprop sys.boot_completed   # → 1 表示启动完成
```

验证：开屏看到 Android Launcher + 状态栏显示电量即可。

### Phase 2: 构建 PokeClaw Debug APK
```bash
cd D:\work\code\PokeClaw
.\gradlew.bat :app:assembleDirectDebug
dir app\build\outputs\apk\direct\debug\
# 应输出 PokeClaw_*.apk (~122MB)
```

### Phase 3: 装到模拟器
```bash
adb -s emulator-5554 install -r -g app\build\outputs\apk\direct\debug\PokeClaw_*.apk
# 后台跑（大文件传输 ~5-15min）
```

验证：`adb -s emulator-5554 shell pm list packages | grep io.agents.pokeclaw` → 应出现

### Phase 4: 配 PokeClaw
1. 启动 PokeClaw（桌面找到图标）：`adb -s emulator-5554 shell am start -n io.agents.pokeclaw/.ui.splash.SplashActivity`
2. 首次启动会自动弹权限请求：点「允许」
3. 授权无障碍：
   - Settings → 权限 → 无障碍 → PokeClaw → 启用
   - Settings → 权限 → 通知监听 → PokeClaw → 启用
4. 配 DeepSeek：
   - PokeClaw → Settings → LLM Config → OpenAI-compatible / Custom
   - baseUrl: `https://api.deepseek.com/v1`
   - model: `deepseek-chat`
   - API key: 从 `TEST_CREDENTIALS.local.md` 复制
5. 启 External Automation（关键步骤！否则 dyq 端无法下发任务）：
   - PokeClaw → Settings → Remote Control → External Automation → Enable

### Phase 5: 设备注册到测试环境 dyq
1. 注入测试环境 baseUrl 到 PokeClaw：
   ```bash
   adb -s emulator-5554 shell run-as io.agents.pokeclaw sh -c \
     "echo 'cloud_base_url=https://saastest.muse.yun' > /data/data/io.agents.pokeclaw/shared_prefs/cloud_url_override.xml"
   # 或更简单：直接修改 KVUtils（需重新打包）或在 Settings UI 找 cloud_base_url 输入
   ```

2. 触发设备注册：重启 PokeClaw 或等 `cloud_enabled=true` 自动注册
3. 验证：登录 `https://saastest.muse.yun`（admin/ZIBsXPIaZPiPjZH）→ 设备管理 → 应看到模拟器设备记录（设备名 `emulator-5554` 类似）

## 5. 风险与回滚

- ⚠️ **无 KVM 沙箱**：当前 WSL2 工作站无 `/dev/kvm`，emulator 起不来。**必须切到有 KVM 的宿主**（BIOS 开启 VT-x + 关 Hyper-V）；否则降级到执行 T1-T4 后停。
- ⚠️ **APK 传输超时**：模拟器安装 `adb install` 通常 30-60s，但首次冷启动 + `-g` 自动授权可能 5min。**用 1800s 超时**。
- ⚠️ **Google 风控**：disposable 账号跑 Gmail/Play 任务可能被风控，**首次需人工过验证**。
- ⚠️ **凭证管理**：本 Goal 含 Google 账号 + dyq admin 密码 + DeepSeek key。**发布公开 fork 前必须净化历史**。
- ⚠️ **任务超时兜底**：240s agent loop 兜底是 BACKLOG P2 已知 bug；T6 短任务不会触发。
