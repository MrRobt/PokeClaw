# 细方向四：页面观察、上下文压缩、经验记忆设计

> 文档编号：PC-RELIABILITY-004  
> 目标：让端侧小龙虾看得准、记得住关键状态、忘掉无用噪声  

---

## 一、页面观察的目标

端侧执行成功率很大程度取决于“当前页面看得准不准”。页面观察不是简单截图，而是生成模型可用的状态摘要。

建议观察对象：

- 当前前台应用包名
- 当前页面标题/核心文本
- 可点击控件列表
- 输入框状态
- 键盘是否弹出
- 最近一次动作后页面是否变化
- 截图路径
- 控件树路径
- 通知/权限/弹窗状态

---

```json
{
  "appPackage": "com.tencent.mm",
  "pageType": "CHAT_DETAIL",

}
```

## 三、上下文分层

端侧上下文必须分层，避免历史噪声把模型带偏。

### 必须常驻

- 用户原始目标
- 当前任务类型
- 当前步骤
- 已完成步骤
- 当前页面摘要
- 安全规则
- 动作白名单
- 完成条件

### 最近保留

- 最近三轮动作
- 最近三轮执行结果
- 最近失败原因
- 最近页面变化摘要

### 可压缩

- 早期工具结果
- 旧截图描述
- 重复控件列表
- 模型解释文本

### 必须丢弃或摘要化

- 大段无关聊天历史
- 过期页面信息
- 重复失败日志
- 与当前任务无关的用户闲聊

## 四、经验记忆

每次任务结束后都要沉淀经验。经验分两类：

### 成功经验

```json
{
  "taskType": "SEND_MESSAGE",
  "appPackage": "com.tencent.mm",
  "successPath": ["OPEN_APP", "LOCATE_CONTACT", "OPEN_CHAT", "INPUT_MESSAGE", "SEND_MESSAGE", "VERIFY_SENT"],
  "keyTargets": ["搜索", "发送"],
  "successRate": 0.86,
  "lastVerifiedAt": "2026-05-21"
}
```

### 失败经验

```json
{
  "taskType": "SEND_MESSAGE",
  "failureType": "TARGET_NOT_FOUND",
  "symptom": "键盘弹出后发送按钮被遮挡或文本不稳定",
  "mitigation": "先隐藏键盘或使用坐标候选区点击",
  "evidenceCount": 5
}
```

---

## 五、经验使用规则

1. 同类任务开始前，优先加载最近成功经验。
2. 当前页面和经验页面不匹配时，不要强套经验。
3. 经验只能建议策略，不能绕过动作校验。
4. 失败经验优先用于避免重复错误。
5. 云端可以下发经验包，端侧负责本地验证有效性。

## 七、页面变化判断

每次动作后要判断页面是否真的变化：

```text
动作前摘要 + 动作后摘要
    ↓
比较前台应用、页面类型、关键文本、控件数量、焦点状态
    ↓
判断：成功变化 / 无变化 / 弹窗打断 / 应用切换 / 未知变化
```

这能避免工具返回成功但页面没有变化的假成功。

---

## 八、验收标准

- 每轮模型输入都包含当前页面摘要。
- 页面摘要能列出可点击目标和输入状态。
- 长任务不会无限追加历史截图描述。
- 任务控制状态不会被上下文压缩误删。
- 同类任务能读取最近成功路径和失败规避策略。
