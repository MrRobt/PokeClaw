# 细方向三：步骤强制、纠偏重试、错误预算设计

> 文档编号：PC-RELIABILITY-003  
> 目标：防止端侧小龙虾跳步、早停、卡死循环，提高长任务完成率  

---

## 一、核心问题

手机任务通常不是单动作，而是多步骤链路。比如“给张三发消息”：

```text
打开微信 → 找到张三 → 进入聊天页 → 输入消息 → 点击发送 → 验证消息出现
```

如果没有步骤约束，模型常见错误是：

- 没打开应用就尝试点击联系人。
- 没输入消息就点击发送。
- 发送按钮没点成功却说完成。
- 搜不到联系人后重复搜索十几次。

---

## 二、步骤计划对象

建议任务计划结构：

```json
{
  "taskType": "SEND_MESSAGE",
  "requiredSteps": [
    {"id": "OPEN_APP", "name": "打开目标应用", "required": true},
    {"id": "LOCATE_CONTACT", "name": "定位联系人", "required": true},
    {"id": "OPEN_CHAT", "name": "进入聊天页", "required": true},
    {"id": "INPUT_MESSAGE", "name": "输入消息", "required": true},
    {"id": "SEND_MESSAGE", "name": "发送消息", "required": true},
    {"id": "VERIFY_SENT", "name": "验证发送成功", "required": true}
  ],
  "terminalStep": "VERIFY_SENT"
}
```

## 三、步骤约束器职责

```text
StepEnforcer
├─ 判断动作是否匹配当前步骤
├─ 记录已完成步骤
├─ 阻止模型提前完成
├─ 阻止模型跳过必要步骤
├─ 为模型生成下一步提示
└─ 在步骤重复失败后触发降级策略
```

关键规则：

1. `FINISH_TASK` 只能在终止步骤完成后通过。
2. 任务计划中的必需步骤不能被模型跳过。
3. 同一步骤连续失败达到阈值后，必须切换策略或上报。
4. 步骤完成必须依赖执行结果或验收证据，而不是模型自称完成。

## 四、错误预算

错误预算分三类：

| 类型 | 示例 | 默认上限 | 超限处理 |
|---|---|---:|---|
| 模型格式错误 | 输出非法动作 | 3 | 切换强约束提示或云端模型 |
| 工具执行错误 | 控件找不到、点击失败 | 3 | 重新观察、换动作策略 |
| 验收错误 | 动作成功但目标未达成 | 2 | 回滚/重试/上报 |

全局还要有最大轮次，例如：

```text
单任务最大模型轮次：12
单任务最大工具调用：20
单步骤最大重试：3
单任务最大时长：120秒
```

## 五、纠偏提示模板

### 格式错误纠偏

```text
上一轮输出不是合法动作。请只输出一个结构化动作，不要输出解释文本。
动作类型必须来自：OBSERVE_SCREEN、TAP_TEXT、TAP_BOUNDS、INPUT_TEXT、PRESS_BACK、WAIT、OPEN_APP、SCROLL、REQUEST_CLOUD_HELP、FINISH_TASK。
```

### 跳步纠偏

```text
你试图执行的动作属于后续步骤，但当前必须先完成：{currentStepName}。
请基于当前页面重新选择一个能推进当前步骤的动作。
```

### 早停纠偏

```text
不能结束任务。当前尚未完成验收步骤：{pendingSteps}。
请继续执行下一步，直到验收器确认成功。
```

### 工具失败纠偏

```text
上一轮动作执行失败：{errorType}，原因：{errorMessage}。
当前页面可见文本：{visibleTexts}。
请换一种动作策略，不要重复同一个失败动作。
```

## 六、降级策略

按成本从低到高：

1. 重新观察页面。
2. 换控件定位方式：文本 → 描述 → 坐标。
3. 返回上一级重新进入。
4. 重启目标应用。
5. 请求云端强模型给下一步建议。
6. 上报失败，等待人工或云端重新下发。

---

## 七、和云端协同

端侧不要把所有失败都吞掉。以下情况必须上报云端：

- 权限缺失。
- 安全拦截。
- 连续三次页面不匹配。
- 同一任务重复失败两次。
- 本地模型无法产出合法动作。
- 任务超时。

上报内容必须包含：

- 任务编号
- 当前步骤
- 已完成步骤
- 失败动作
- 错误类型
- 截图/控件树证据
- 端侧建议：重试/改派/人工确认/放弃

---

## 八、验收标准

- 未完成必要步骤时，模型输出完成会被拦截。
- 同一失败动作不会无限重复。
- 超过错误预算后任务进入明确终态。
- 失败回执能说明卡在哪一步。
- 云端能基于回执复盘失败原因。
