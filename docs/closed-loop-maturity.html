04

命门 · 阻断「自主 + 安全迭代」的关键缺口

H1感知决策大脑是空的

云手机 /control 是坐标驱动——「该点哪个 (x,y)」没人算。路线 A 的 ai_decide 拿到假决策；端侧路线 B 目前靠无障碍文本、遇自绘/游戏/反无障碍即瞎。

→ 把 YOLO 权重跑成实时检测/多模态决策服务，替掉写死数据

H2模型安全上端缺门禁

新策略/模型直接置为 current，无影子对比、无质量门禁；飞轮转，但转得不安全，新模型可能直接上端翻车。

→ 影子跑达标才晋升（YOLO 侧已有 gate，训练要接真的）

H3降级链中间一级没接

「端侧处理不了→云端大模型接手」没接进降级流，直接跳人工；升级信号协议端云对不齐；人工接管通知无消费者。

→ 端侧云执行体升级为完整 Agent + 补 escalate 协议 + 接管消费者

H4端云链路真后端 E2E 存疑

边界文档说 mock-only/从未真后端验证；另一份材料声称 :48081 E3 往返。两者打架，决定采集数据/任务能不能真打通。

→ 一锤定音核实真后端 register→下发→回传全环
